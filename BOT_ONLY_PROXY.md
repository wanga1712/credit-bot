# Настройка прокси только для бота (без системного VPN)

## Вариант 1: SSH туннель через ноутбук (РЕКОМЕНДУЕТСЯ)

Если ваш ноутбук может подключиться к Telegram (не в России или использует VPN):

**На ноутбуке (Windows PowerShell):**
```powershell
# Создайте SOCKS5 прокси на порту 1080
ssh -D 1080 -N wanga@100.122.104.106
```

**На сервере добавьте в `.env`:**
```bash
TELEGRAM_PROXY=socks5://127.0.0.1:1080
```

**Проблема:** Ноутбук должен быть включен. Для постоянной работы можно настроить автозапуск.

## Вариант 2: WARP в режиме прокси (через дополнительный прокси-сервер)

WARP не поддерживает режим только прокси напрямую, но можно использовать обходной путь:

1. Установите WARP на отдельном сервере/контейнере
2. Настройте на нем SOCKS5 прокси
3. Используйте его IP в `.env`

Это сложнее и требует дополнительный сервер.

## Вариант 3: Бесплатный прокси за пределами России

Найдите бесплатный прокси (не из России):

1. Сайты со списками:
   - https://www.proxy-list.download/
   - https://free-proxy-list.net/
   - Фильтруйте: страна != Russia

2. Добавьте в `.env`:
   ```bash
   TELEGRAM_PROXY=http://proxy-ip:port
   # или для SOCKS5:
   TELEGRAM_PROXY=socks5://proxy-ip:port
   ```

**Проблемы:** Ненадежно, медленно, нужно часто менять.

## Вариант 4: Использование другого сервера как прокси

Если у вас есть другой сервер за пределами России:

1. На том сервере установите прокси (tinyproxy, squid, или SSH туннель)
2. Используйте его IP в `.env`:
   ```bash
   TELEGRAM_PROXY=http://external-server-ip:port
   ```

## Рекомендация

**Для постоянной работы:** Используйте **Вариант 1** (SSH туннель через ноутбук), но настройте автозапуск.

**Для тестирования:** Попробуйте **Вариант 3** (бесплатный прокси).

## Настройка автозапуска SSH туннеля (для Варианта 1)

Чтобы SSH туннель работал постоянно:

**На ноутбуке создайте задачу в планировщике Windows:**
- Программа: `C:\Windows\System32\OpenSSH\ssh.exe`
- Аргументы: `-D 1080 -N wanga@100.122.104.106`
- Запускать при входе в систему

Или используйте autossh на сервере (но это требует SSH доступ с сервера на ноутбук).
